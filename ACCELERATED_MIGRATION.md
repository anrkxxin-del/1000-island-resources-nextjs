## 🔄 DNS 切换步骤

### Step 1: 准备新 DNS 记录

在 Vercel 中获取指向地址：
```
Vercel Dashboard → Project Settings → Domains
→ 找到 "Nameservers" 或 "CNAME"
```

### Step 2: 更新域名 DNS

去你的域名注册商（如 Namecheap、Godaddy 等）：

```
当前 DNS:
www.1000-island.com → NiceBox (192.168.x.x)

改为：
www.1000-island.com → Vercel (cname.vercel-dns.com)
```

### Step 3: 等待 DNS 生效

```
时间：通常 2-48 小时（初始 5-30 分钟）

验证：
curl -I https://www.1000-island.com
# 应该返回 200 OK（Vercel）
```

### Step 4: 配置 301 重定向

如果旧站点有重要 URL，设置 301 重定向到新站：

```
# 在 Next.js 的 next.config.js 中配置
redirects: async () => {
  return [
    { source: '/about', destination: '/resource', permanent: true },
    { source: '/course/:path*', destination: '/resource/teacher/:path*', permanent: true },
    // ... 其他重定向
  ]
}
```

## 🎯 内容生成加速（1 天 20 页）

**不要一篇篇生成，用批量方式**：

### Prompt 模板（生成 10 篇一次性）

```
我需要一次性生成 10 篇英文文章。
使用下面的 JSON 输入来定义每篇文章：

{
  "articles": [
    {
      "title": "How to Choose an English Teacher",
      "slug": "how-to-choose",
      "category": "teacher",
      "keywords": ["english teacher", "online tutor", "philippines"],
      "focus": "practical guide with comparison tables"
    },
    {
      "title": "Top 10 Online English Tutors in Philippines",
      "slug": "top-10-tutors",
      "category": "teacher",
      "keywords": ["online tutors", "philippines", "best teachers"],
      "focus": "detailed review and ranking"
    },
    ... (8 more articles)
  ]
}

对每篇文章，生成：
1. 完整的 frontmatter（含 title, slug, category, keywords, date）
2. 1200-1500 字的原创内容
3. 至少 1 个对比表格
4. 至少 3 个内部链接建议
5. 至少 3 个 FAQ

格式：完整的 .mdx 文件，用 --- 分隔

返回所有 10 个完整文件。
```

**结果**: 1-2 小时完成 20 页（而不是 20 小时）

## ⚠️ 可能遇到的问题

### 问题 1: 内容生成慢（Day 3-5）
**解决**: 用批量 prompt，1-2 小时完成 20 页

### 问题 2: SEO 没有立即排名
**预期行为**：这很正常。SEO 需要 2-4 周。不会影响切换决策。

### 问题 3: 用户反馈说页面样式有问题
**处理流程**：
1. 记录具体问题
2. 修复（改 CSS 或内容）
3. Git push → Vercel 自动部署
4. 通知用户已修复

### 问题 4: 咨询数量下降
**分析**：
- 是导航链接不明显？
- 是新站上的 CTA 不够强？
- 是用户找不到联系方式？
→ 快速修复，重新测量

### 问题 5: DNS 切换后 404 错误
**原因**：旧 URL 重定向配置缺失
**解决**：添加 301 重定向规则，再次部署

## 📞 切换 SOP（Standard Operating Procedure）

### 切换前 1 小时
```
1. 确认所有前置条件满足
2. 通知团队：即将进行域名切换
3. 打开 Vercel Dashboard 和 GA 的实时监控
4. 准备好回滚方案
```

### 切换时刻
```
1. 登录域名注册商
2. 修改 DNS 记录（从 NiceBox → Vercel）
3. 记录切换时间戳
4. 等待 5-30 分钟首次生效
```

### 切换后 1 小时
```
1. 验证主域名访问正常
2. 检查 Vercel 构建是否进行中
3. 检查 GA 流量是否正常
4. 扫描收件箱是否有错误报告
```

### 切换后 24 小时
```
1. 检查 Google Search Console 是否显示索引变化
2. 验证 301 重定向是否工作
3. 检查核心功能（表单提交、链接等）
4. 汇总用户反馈
```

### 切换后 7 天
```
1. 检查所有关键指标是否稳定
2. 比较新旧流量数据
3. 检查 SEO 变化（排名、展现次数）
4. 评估迁移成败
```
