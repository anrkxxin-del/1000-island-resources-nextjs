# 平行迁移战略（Parallel Migration）

## 🎯 核心策略

**不替代，共存。** 在完全验证新系统前，保持两个独立的网站运行。

```
时间轴：

现在 (Week 1-4)
├─ NiceBox 官网：保留、继续运营、获取转化
├─ Next.js 新站：独立开发、积累内容
└─ 域名配置：www.1000-island.com = 旧站

第二阶段 (Week 5-8)
├─ 新站稳定：SEO 表现验证、流量开始增长
├─ 官网首页：添加「资源中心」导航 → 链向新站
└─ 监控指标：转化率、用户行为、技术稳定性

第三阶段 (Week 9-12)
├─ 新站成熟：20+ 个内容页、SEO 排名稳定
├─ 主域切换：考虑 www.1000-island.com → 新 Next.js 站
└─ 旧站存档：NiceBox 逐步归档或作为「关于」页面

```

## 📍 部署架构

```
用户访问
    │
    ├─ www.1000-island.com (当前：NiceBox)
    │  ├─ 首页、产品页、联系我们
    │  ├─ 品牌、课程、报名
    │  └─ [计划] 导航栏添加「资源中心」链接
    │
    └─ resources.1000-island.com (新：Next.js + Vercel)  ← 或 /resource 子路径
       ├─ /teacher - 外教指南
       ├─ /study-abroad - 游学指南
       ├─ /faq - 常见问题
       ├─ /comparison - 对比页面
       └─ SEO 内容聚合 (AI 优化)
```

## 🔧 实施步骤

### Phase 1: 独立开发（Week 1-4）

#### 目标
- ✅ Next.js 站点完全可用
- ✅ 20 个高质量内容页面上线
- ✅ SEO 基础设施完备（Sitemap、Schema、Meta）
- ✅ 内部测试无误

#### 工作项
```
□ 项目初始化 + GitHub 部署
□ 生成 10 个教师指南页面
□ 生成 10 个游学指南页面
□ 配置 SEO metadata 自动化
□ 本地测试（功能、性能、SEO）
□ 发布到 Vercel
□ 设置 robots.txt: Disallow all（防止被索引）
```

#### 流量期望
- 新站：0（禁止索引中）
- 旧站：照常运营

### Phase 2: 灰度发布（Week 5-8）

#### 目标
- ✅ 从官网导入部分用户到新站
- ✅ 验证新站的转化率和用户体验
- ✅ 收集反馈并持续优化

#### 工作项

**在 NiceBox 官网添加导航**：
```html
首页导航栏添加链接：
「资源中心」→ https://resources.1000-island.com/
    或
「学习指南」→ https://1000-island.com/resource/
```

**更新 robots.txt**（启用索引）：
```robots.txt
User-agent: *
Allow: /

Sitemap: https://resources.1000-island.com/sitemap.xml
```

**添加内容**：
- 新增 20 个内容页面（第二批）
- 优化已有内容的内部链接
- 实现自动 sitemap 生成

#### 监控指标

使用 Google Analytics 在两个站点追踪：

**新站 (Next.js)**：
- 日均 PV（目标：100+）
- 平均停留时长（目标：> 2 分钟）
- 跳出率（目标：< 60%）
- 页面转化率（指向官网的链接点击）

**旧站 (NiceBox)**：
- 保持转化率不变或提升（来自资源中心的转化）
- 用户来源分析

#### 流量期望
- 新站：10-100 PV/day（SEO 前期）
- 旧站：保持稳定或微增（得益于新站的反向链接）

### Phase 3: 全面切换（Week 9-12）

#### 条件（满足以下 3 个条件才切换）

1. **技术稳定性**
   - [ ] 新站无 500 错误（Vercel 构建成功率 100%）
   - [ ] 响应时间 < 500ms (Core Web Vitals 绿色)
   - [ ] SSL 证书有效

2. **SEO 表现**
   - [ ] 在 Google Search Console 中显示 > 50 个关键词排名
   - [ ] 至少 3 个关键词进入前 10 名
   - [ ] 有机流量 > 500/month

3. **业务表现**
   - [ ] 新站带来的转化 ≥ 旧站的 10%
   - [ ] 用户满意度评分 ≥ 4/5
   - [ ] 支持团队反馈正面

#### 切换步骤

**步骤 1: 准备 (Day 1)**
```bash
# 旧站导出所有内容（备份）
# 新站最终检查：所有页面、链接、表单
```

**步骤 2: DNS 切换 (Day 2)**

如果选择完全迁移到新站：

```dns
# 当前配置：
www.1000-island.com → NiceBox (192.168.x.x)

# 改为：
www.1000-island.com → Vercel (Next.js)
# Vercel 提供的 DNS 指向

# 时间：2-24 小时生效
```

**步骤 3: 301 重定向 (Day 3-7)**

从 NiceBox 设置 301 重定向到新站对应页面：
```
/product → /resource/teacher/
/about → /resource/ 或新的关于页面
/faq → /resource/faq/
```

**步骤 4: 监控 (Day 7-30)**

```
□ Google Search Console: 重新提交 sitemap
□ Bing Webmaster: 通知索引更新
□ Analytics: 监控流量、跳出率、转化
□ 错误日志: 检查 404、500 等异常
□ 用户反馈: 邮件、客服反馈
```

#### 流量期望
- 新站：1000+ PV/day（来自 SEO + 直接流量）
- 旧站：逐步归档或只保留特定页面

## 💰 成本预估

| 项目 | 成本 | 备注 |
|------|------|------|
| GitHub 账户 | 免费 | 公开仓库 |
| Vercel 部署 | 免费（初期） | Pro 计划 $20/月（可选） |
| 域名 / DNS | $0 | 使用现有 1000-island.com |
| Claude Code 调用 | 按 API 使用 | 内容生成脚本 |
| **总计** | **< $50/月** | 初期无需付费 |

## ⚠️ 风险和应对

| 风险 | 概率 | 应对 |
|------|------|------|
| 新站 SEO 表现不理想 | 中 | 保持旧站，继续优化内容 |
| 用户困惑（两个网站） | 低 | 官网清楚说明：官网 vs 资源站 |
| 转化率下降 | 低 | 新站有明确的内链回官网 |
| Vercel 宕机 | 极低 | 有备用重定向方案，自动故障转移 |
| DNS 切换延迟 | 中 | 计划在流量低谷（如午夜）执行 |

## 总结

**这个策略的好处：**

1. **零风险** - 旧系统一直在线，任何时候可以回滚
2. **数据驱动** - 基于实际指标做切换决策
3. **渐进优化** - 有时间逐步完善新系统
4. **流量积累** - 通过导航链接逐步导入流量
5. **技术验证** - 充分时间发现和修复问题

**时间预估：3-4 个月完全迁移，过程中零中断。** ✅
